## Heaer

We dont need Call 101 in the header 

Header format can use a patient type -> emergency -> protocol format

For example:

ADULT - ASTHMA - ACUTE ATTACK

## Flow chart

Flow charts look good but the labels (for YES / NO) are squished. 

Potential solutions are:

- Write as Y/N
- Use tick / x symbols im green and red

## Text 

Clarity is key. 

In the asthma response protocol for example we don't need the steps itemised (step 1, step 2) as it increases cognitive load. 


## Missing Links

Sometimes we have links that could be denoted using feint lines.

For example:

The yes response for Step 3 (provide the inhaler) then instructs to proceed to step 4. That can be denoted with a feint diagonal line leading to line 4.

## End of Path

Sometimes, a pathway reaches its conclusion

We can use two underlines to denote that

## Icons

The selective use of icons can make the information more digestible. The key is to use them judiciously.

For the administer inhaler step an icon of an MDI inhaler could be used. 

When a pathway includes an instruction to call a body (like MDA / 101) we should use a common, consistent logo. 

See: /icons for logos by provider. This should be in addition to the text. 

## Minimising cognitive load

The key to designing effective flowcharts is assumig that the person reading them will be doing so in a state of acute stress. 

For that reason, clarity and simplicity are essential - as is visual communication. 

## Page Numbers

Increase size 50% and use neon yellow as backgorund cover 

## Identifiers 

For ease of maintenance, each SOP should have a unique ID. Use a simple three digit numeric format. 

For example Asthma Attack - Adult (1)
